I need a feature store pattern with separate online and offline paths. 4. **Training pipeline**: Weekly batch job that produces versioned model artifacts, validates before promotion, and supports rollback. 5. **Model registry**: Track versions and metadata, enable rollback. 6. **Rollback**: Blue/green or canary deployment with N stored versions switchable via config or feature flag. Now I'm mapping out the bounded contexts and how they interact.
