Full audit logging of every field access 4. Offline capability when KMS is unavailable 5. Zero-downtime key rotation every 30 days Let me check for contradictions: - Requirement 2 (50ms P99) vs Requirement 3 (audit every field access) — logging every field access adds latency - Requirement 1 (CMK) vs Requirement 4 (offline when KMS unavailable) — if keys are customer-managed externally, how do you decrypt offline? - The core tension here is that CMK typically means keys live in an external KMS, but offline mode requires local key access, which creates a security trade-off. And achieving 50ms P99 while doing full field-level audit logging is extremely challenging. These aren't necessarily impossible to resolve with the right architectural choices, but they do require significant trade-offs and business input on what's actually acceptable. The CMK plus offline requirement is the real contradiction — I can work around it with local key caching and envelope encryption, but that changes what "customer-managed" actually means.
