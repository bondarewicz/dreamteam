# Follow-up: 2026-03-23 Session

## What was done

### 1. Draft eval artifact generation (commands/team.md + evals/draft-template.md)
Every team session now auto-generates draft eval scenario files capturing agent input/output pairs. Drafts go to `evals/<agent>/drafts/draft-YYYY-MM-DD-HHMM-<topic>-NNN.md` and are excluded from eval-run.sh by directory depth + filename pattern. Promotion to the eval suite is a manual move + rename.

### 2. Agent filter chips in eval report (scripts/eval-report.sh)
Focus section now has clickable filter chips (All, Bird, Mj, Shaq, Kobe, Pippen, Magic). Filters action items, change chips, and agent sections. Works offline, print-safe.

### 3. Coach K output validation + retry (commands/team.md)
Coach K now validates every agent's JSON output after completion. Non-compliant output (fences, prose) triggers up to 2 retries with a stricter preamble. Falls back to manual strip after 2 failures.

### 4. Bird spec structured output improvements (agents/bird.md)
- Output contract: machine-to-machine framing, explicit first/last character rules
- Escalation type classification: clear definitions with table, type priority guidance
- Stop conditions: contradiction no longer forces empty acceptance_criteria
- Analysis workflow: classify escalation type BEFORE analysis

### 5. Grader `contains` semantics (scripts/eval-graders.sh + 5 scenarios)
New `contains` grader check: for wildcard paths, verifies AT LEAST ONE value matches (vs `equals` which requires ALL). Applied to escalation type checks in scenarios 10, 15, 18, 19, 20.

## Key learnings

- Prompt engineering for structured JSON output plateaus at ~70% pass@1 for complex scenarios. Anthropic's docs confirm: "prompting alone is unreliable."
- The `contains` grader semantics were the breakthrough — they accept Bird's substantively correct analysis even when it adds extra escalation types.
- Single-trial eval measurements are too noisy for iterative development. Always use `--trials 3` when measuring spec changes.
- Bird's dominant failure pattern: adding `missing_context` escalation items alongside other types. This is a deep model tendency that prompt engineering cannot fully suppress.
