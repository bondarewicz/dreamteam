§ 02 — THE PLAYBOOK

How /team
runs the floor.

One command. Three modes. Coach K suggests the lineup, runs the handoffs, and hands every ship decision back to you.

I

Quick Fix

Subagents · in-session · low-cost

Bug fixes, small features, focused changes. Bird → Shaq → Kobe → Magic, inside one Claude Code session. You see every checkpoint. Coach K drafts the git commands; you run them.

II

PR Review

Parallel subagents · local output only

Three reviewers hit the diff at once. Coach K fetches PR data with read-only gh, fans out to Bird, MJ, and Kobe, synthesizes the verdicts, and writes docs/PR-NN-review.md. Nothing is ever posted to GitHub — you decide what to say.

III

Full Team

Agent Teams · 6 parallel sessions · checkpointed

New features, architecture changes, multi-file work. Six independent sessions, five phases. After analysis, Coach K stops and shows you the plan — Shaq doesn't touch a line of code until you approve it. Every phase checkpoints to disk.

◆ What's in the booth

01
Run evals live

Kick off a run from the browser. Results stream via Server-Sent Events as Coach K scores each scenario — no refresh, no polling.

02
Author with AI-generated graders

Write a scenario; the workbench drafts graders for you. Validate, generate, dry-run, promote — the full authoring loop lives in the UI.

03
Trace every turn

Drill into any run to see the agent's full trace — inputs, tool calls, thinking, outputs. The receipts, not just the score.

04
Auto-migrates on first boot

Existing JSON results are imported into a local SQLite database the first time you start the server. No manual migration step.

05
Local-first by design

Your eval data never leaves the machine. One command to boot: cd web && bun run start. That's the whole install.

§ 04 — THE INSTALLATION

Install once.
Play everywhere.

One command. The repo is the source of truth. Your MCP servers survive the offseason.

01

Clone and run the installer

$ git clone https://github.com/bondarewicz/dreamteam
$ cd dreamteam && bun scripts/install.ts

The installer backs up existing files, then syncs agents, commands, scripts, and MCP servers into ~/.claude/.

02

Restart Claude Code

A fresh session picks up every agent, every command, every MCP server — ready to go.

03

Call a play

$ claude
> /team build a discount engine for bulk orders

◆ On the stat sheet

You have final say
No agent ever commits, pushes, merges, or ships. Coach K drafts the git commands, pauses at every checkpoint, and waits for you. Human-in-the-loop is non-negotiable.
Source of truth
Every agent, command, and script is tracked in the repo. install.ts syncs to ~/.claude/ — no drift, no stale copies.
MCP server persistence
Tracked in scripts/mcp-servers.json. The installer merges into ~/.claude.json non-destructively, so your MCP config survives a clean reinstall.
Git worktree isolation
Concurrent /team sessions run in isolated worktrees — no branch collisions, no half-merged state.
125 eval scenarios
20–25 per agent, auto-scored with Coach K. Run --trials 3 before shipping any agent spec change.
Web dashboard
A local Bun server serves eval results as a browsable report. No auth, no network hops.
Miro integration
Agents can read boards and build diagrams — event storming, service decomposition, domain catalogs — without leaving Claude Code.
